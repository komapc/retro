# Data Directory Reference

This directory contains all input definitions and pipeline output for Factum Atlas.
Runtime artifacts (`vault2/`, `raw_ingest/`, `atlas/`, `progress.json`) are gitignored — they are generated by the pipeline.

---

## Tracked (in git)

### `data/events/{id}.json`
One file per event. The pipeline ingests articles and extracts predictions for each event.

```json
{
  "id": "C05",
  "name": "Iran launches direct missile attack on Israel",
  "outcome": true,
  "outcome_date": "2024-04-14",
  "search_keywords": ["מתקפה איראנית ישירה", "Iran missile attack Israel"],
  "llm_referee_criteria": "Must focus on the April 14 multi-drone/missile barrage."
}
```

| Field | Description |
|---|---|
| `id` | Unique event ID (letter prefix = category, number = index) |
| `name` | Human-readable event description |
| `outcome` | `true` = event happened, `false` = did not happen, `null` = unresolved |
| `outcome_date` | Date the event resolved (ISO 8601) |
| `search_keywords` | List of keywords for GNews/DDG/CDX search. Hebrew + English. |
| `llm_referee_criteria` | Prompt fragment given to the LLM to judge relevance |
| `predictive_window_days` | (optional) How many days before outcome_date to search. Default: 14 |

**Event ID prefixes:**
- `A` — Israeli politics / elections
- `B` — Gaza / Hamas conflict
- `C` — Iran / regional escalation
- `D` — Israeli economy
- `E` — Global events (Trump, Fed, etc.)
- `F` — Society / religion
- `G` — Tech / AI

### `data/sources/{id}.json`
One file per news source. Controls which outlets the pipeline ingests.

```json
{
  "id": "toi",
  "name": "Times of Israel",
  "url": "https://www.timesofisrael.com",
  "country": "IL",
  "language": "en",
  "type": "newspaper",
  "api_provider": null,
  "ingestion_status": "pending",
  "political_lean": null,
  "economic_lean": null,
  "notes": "English-language Israeli outlet"
}
```

| Field | Description |
|---|---|
| `id` | Unique source ID (used as directory name in raw_ingest/) |
| `url` | Base URL — domain extracted for CDX/GNews search |
| `language` | ISO 639-1 code. `he` sources use Hebrew GNews locale |
| `ingestion_status` | `pending` / `active` / `disabled` |

## Generated at runtime (gitignored)

### `data/progress.json`
Tracks pipeline state for every (event, source) cell.

```json
{
  "cells": {
    "C05:toi": {
      "event_id": "C05",
      "source_id": "toi",
      "status": "done",
      "prediction_count": 4,
      "error": null
    }
  }
}
```

**Cell status state machine:**

```
pending → done          (articles found + predictions extracted)
pending → no_predictions (articles found but gatekeeper/extractor returned 0 predictions)
pending → failed        (LLM returned invalid JSON after all retries)
pending → running       (currently being processed)
```

To re-process `no_predictions` cells: reset their status to `pending` and rerun the pipeline.

---

### `data/raw_ingest/{source_id}/{event_id}/article_NN.json`
Raw articles fetched by `gnews_ingest.py` before LLM processing.

```json
{
  "url": "https://www.timesofisrael.com/...",
  "headline": "Iran fires drones and missiles at Israel",
  "text": "Full article body...",
  "published_at": "2024-04-13",
  "author": "ToI Staff"
}
```

Articles under 500 chars are filtered (paywalled stubs). Liveblogs are skipped.

---

### `data/vault2/articles/{hash}.json`
Deduplicated article store. Key = SHA-256 of article text. Same article fetched from multiple events is stored once.

### `data/vault2/extractions/{hash}_{event_id}_v1.json`
LLM extraction result for a specific article × event pair. Cached — reused if article is seen again for the same event.

```json
{
  "extraction": {
    "predictions": [
      {
        "quote": "Iran will retaliate within days",
        "claim": "Iran will strike Israel within days",
        "stance": 0.85,
        "certainty": 0.9,
        "sentiment": -0.3,
        "hedge_ratio": 0.1,
        "time_horizon": "days",
        "time_horizon_days": 3,
        "prediction_type": "binary",
        "source_authority": 0.8
      }
    ]
  },
  "prompt_version": "v1",
  "run_date": "2026-03-22T08:14:00"
}
```

---

### `data/atlas/{event_id}/{source_id}/entry_{hash8}.json`
Rendered atlas entries — one file per article that produced predictions. These feed `render_atlas.py` to build `factum_atlas.html`.

---

## Pipeline flow

```
data/events/ + data/sources/
        ↓
  gnews_ingest.py          → data/raw_ingest/
        ↓
  orchestrator.py          → data/vault2/ + data/atlas/ + data/progress.json
        ↓
  render_atlas.py          → factum_atlas.html
```
